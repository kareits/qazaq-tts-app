# Проверка казахской нормализации чисел (ревью)

Файл сгенерирован автоматически из нашего кода (backend). Ниже — как программа
превращает цифры, даты, время и т.п. в слова для синтеза казахской речи (TTS).

## Задача для проверяющего (ИИ-агента / носителя)

Ты — эксперт по **казахскому языку (кириллица)**. Проверь **грамматическую
корректность** каждой формы ниже: количественные и порядковые числительные,
падежные окончания (сингармонизм + ассимиляция согласных), чтение дат, времени,
диапазонов и телефонов. Для КАЖДОЙ ошибки укажи: (1) вход, (2) наш вывод,
(3) правильную форму, (4) короткое правило/причину. Особое внимание — краевым
формам (заимствования «минут/сағат», порядковые жиырмасыншы/қырқыншы/елуінші,
чтение десятичных). Если форма верна — не перечисляй.

## Результат проверки

Ниже перечислены только ошибки и спорные места, где текущий вывод нужно
исправить. Остальные формы из таблиц 1–5 грамматически корректны для TTS.

### Ответы по разделу 6

- **Творительный падеж**: точное правило для выбора -мен/-бен/-пен: после гласных
  и сонорных согласных (й, л, м, н, ң, р, у) — **-мен**; после звонких шумных
  согласных (б, в, г, ғ, д, ж, з) — **-бен**; после глухих согласных — **-пен**.
  Поэтому: «онмен», «жүзбен», «мыңмен».
- **Порядковые с изменением основы**: «жиырмасыншы», «қырқыншы», «елуінші»,
  «оныншы», «мыңыншы» — корректные формы.
- **Заимствования минут/сағат**: «минутта», «сағаттан», «сағатқа», «сағатта»
  корректны. Оба слова в этих формах берут заднерядные окончания; финальный т
  требует глухих алломорфов -та/-тан/-қа.
- **Десятичные**: конструкция «үш бүтін жүзден он төрт» корректна. Модель
  «бүтін + знаменатель в исходном падеже + числитель» подходит для чтения
  десятичных: «оннан бес», «жүзден он төрт», «мыңнан жүз жиырма бес».
- **Диапазон**: рамка «-дан/-ден ... -ға/-ге дейін» грамматически правильна для
  числового диапазона. Для годов лучше склонять «жыл»: «екі мың жиырмасыншы
  жылдан екі мың жиырма төртінші жылға дейін».
- **Время**: «сағат он төрт отызда», «сағат тоғыздан жиырма бірге дейін»,
  «сегіз қырық бесте» естественны для нейтрального чтения времени в TTS.
  Падежный суффикс на последнем слове времени корректен.
- **Дата**: «бесінші мамыр екі мың жиырма төртінші жыл» корректно: день и год
  читаются порядковыми, месяц остается названием месяца.

### Рекомендации ИИ-агенту по доработкам

Доработать морфологическое ядро `backend/app/services/num2words_kk.py` и затем
подключить исправления в `backend/app/services/text_normalizer.py`, не ломая
текущие публичные функции `cardinal_kk`, `ordinal_kk`, `digits_kk`,
`decimal_kk`.

1. Исправить падежную классификацию в `attach_case`.

   Сейчас `_NASALS = "мнң"` и `_INSTRUMENTAL = {"voiceless": "пен", "nasal":
   "бен", "_": "мен"}` дают ошибочные формы «онбен», «мыңбен», а «жүзмен»
   выходит из default-класса. Нужно разделить финальные согласные так:

   - глухие согласные → `-пен`;
   - звонкие шумные `б в г ғ д ж з` → `-бен`;
   - гласные и сонорные `й л м н ң р у` → `-мен`.

   Ожидаемые тесты:

   - `attach_case("он", "instrumental") == "онмен"`;
   - `attach_case("жүз", "instrumental") == "жүзбен"`;
   - `attach_case("мың", "instrumental") == "мыңмен"`;
   - `attach_case("бес", "instrumental") == "беспен"`;
   - `attach_case("бір", "instrumental") == "бірмен"`.

2. Исправить обработку финального `у` в падежах.

   Сейчас `у` находится в `_VOWEL_ENDINGS`, поэтому «елу» получает ошибочные
   формы «елуні», «елунің». Для числительного «елу» нужны:

   - `елуді`;
   - `елудің`;
   - при этом сохранить `елуге`, `елуде`, `елуден`, `елумен`.

   Практически: убрать `у` из класса обычных гласных для выбора
   accusative/genitive и обрабатывать его как сонорный/полугласный финал.

3. Добавить регрессионные тесты в `backend/tests/test_num2words_kk.py`.

   Минимальный набор:

   - творительный: `онмен`, `жүзбен`, `мыңмен`, `беспен`, `бірмен`;
   - «елу» во всех падежах, особенно `елуді`, `елудің`;
   - контроль обычных гласных основ: `жетіні`, `жетінің`, `жиырманы`,
     `жиырманың`.

4. Отдельно доработать контекстные правила в `text_normalizer.py`.

   Эти ошибки не чинятся одним `num2words_kk.py`, потому что зависят от
   соседних слов и формата токена:

   - даты с `күні`: не «жыл күні», а рамка `<год>-дың <день> <месяц> күні`;
   - обстоятельственные даты без следующего слова «жыл»: `31.12.2025` в
     предложении → `... екі мың жиырма бесінші жылы`;
   - день перед склонённым месяцем: `10 шілдеде` → `оныншы шілдеде`;
   - минуты `01`–`09`: `14:05` → `он төрт нөл бес`;
   - диапазоны времени с `аралығында`: `9:30–17:45 аралығында` →
     `тоғыз отыз бен он жеті қырық бес аралығында`;
   - коды/PIN с ведущими нулями: `Код 12345`, `ПИН 0000` читать по цифрам;
   - падеж после валюты: `2500 ₸-ден` → `екі мың бес жүз теңгеден`;
   - знак номера: `№5 кабинет` → `бесінші кабинет`;
   - версии: `2.0` → `екі нүкте нөл`, а не десятичная дробь.

   Для этих случаев можно либо добавить маленькие helper-функции в
   `num2words_kk.py` (`clock_time_kk`, `version_kk`, `currency_kk`), либо
   оставить ядро чистым и реализовать контекстные правила в `text_normalizer.py`
   через уже существующие `cardinal_kk`, `ordinal_kk`, `digits_kk`,
   `attach_case`.

## 6. Формы, в которых мы особенно не уверены — проверь пристально

- **Творительный падеж (-мен/-бен/-пен)** — наименее уверенная таблица. Проверь
  особо: наш вывод даёт «онбен», «жүзмен», «мыңбен» — верны ли они или должно быть
  «онмен», «жүзбен», «мыңмен»? Сформулируй точное правило выбора -мен/-бен/-пен.
- Порядковые с изменением основы: **жиырмасыншы** (20), **қырқыншы** (40),
  **елуінші** (50), **оныншы** (10), **мыңыншы** (1000).
- Падежи заимствований: **минут/сағат** — «минутта», «сағаттан», «сағатқа»,
  «сағатта» (верна ли гармония/ассимиляция?).
- Десятичные: «3,14» → «үш бүтін жүзден он төрт» (корректна ли конструкция
  «бүтін» + аблатив знаменателя «жүзден»?).
- Диапазон: «2020–2024» → «…дан …ға дейін» (правильная ли рамка «-дан … -ға
  дейін»?).
- Время: «14:30» → «он төрт отыз» (без слов «сағат/минут»); падеж на последнем
  слове («14:30-да» → «отызда»). Естественно ли это?
- Дата: «05.05.2024» → «бесінші мамыр екі мың жиырма төртінші жыл» (день —
  порядковый; год — порядковый + жыл).
