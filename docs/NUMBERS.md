# iSoyle — план нормализации чисел и дат (казахский)

Преобразование цифр, процентов, валют, дат и времени в **словесную** форму для
синтеза. Главная сложность — казахская морфология: количественные vs порядковые
числительные (екі → екінші) и падежные окончания на числах (10-ға → онға).

> **Правило работы**: этот документ — поэтапный план. **Переход к следующему
> этапу — только после явного одобрения.** Отмечаем прогресс здесь по ходу.

---

## Архитектурные решения (зафиксировано)

1. **Где расширять**: только на **этапе синтеза**, посегментно. `split_sentences`
   и `/api/split` продолжают работать по исходному тексту → пользователь видит
   «2024», офсеты и подсветка не ломаются; модель получает «екі мың жиырма төрт».
2. **Куда встроить**: функция `expand_numbers_kk(text) -> str` в
   `text_normalizer.py` (единое место нормализации, правило CLAUDE.md), вызывается
   в `tts_service` для каждого сегмента **перед** передачей в движок.
3. **Ядро** — отдельный модуль `app/services/num2words_kk.py` (чистые функции,
   без внешних зависимостей; `num2words` для казахской морфологии не подходит).
4. **Десятичный разделитель — и запятая, и точка**; в казахской кириллице
   **чаще запятая** («3,14»). Правило разбора без ложных срабатываний:
   десятичное — это `цифра` + вплотную `,`/`.` + вплотную `цифра` (без пробелов).
   «3,14»/«3.14» → десятичное; «3, 14» (пробел) → обычная пунктуация, не число.
   Разрядный разделитель в KZ — **пробел** («1 000 000»), не запятая/точка.
   > Краевой случай: «1,000» из иностранного ввода неоднозначно (тысяча vs 1,0) —
   > по умолчанию трактуем `,`/`.` как десятичный; при необходимости добавим
   > эвристику «ровно 3 цифры после разделителя + ещё группы → разряды».
5. **Предусловие сплиттера**: `split_sentences` не должен считать **точку**
   (`.`/`…`) терминатором между цифрами (десятичные «3.14», даты «05.05.2024») —
   иначе токен числа распадётся. Запятая терминатором и так не является, но при
   парсинге числа принимается как десятичный разделитель. Это часть Этапа 1.
6. **Комментарии/докстринги — на английском** (правило проекта); пользовательские
   строки/ошибки — без изменений.

## Этап 1 — Количественные числительные + фикс сплиттера ▶️ (следующий)

**Цель**: базовое чтение целых чисел, процентов и валюты; сплиттер не рвёт числа.

**Объём**:
- [ ] Фикс `split_sentences`: точка/… между цифрами — не терминатор (десятичные,
      `dd.mm.yyyy`). Тесты на «3.14», «05.05.2024», «Цена 1.5.» (конец предложения).
- [ ] Десятичный разделитель — **`,` и `.`** (запятая приоритетна для KZ),
      только вплотную между цифрами. Тесты: «3,14», «3.14», «3, 14» (НЕ число).
- [ ] `num2words_kk.cardinal_kk(n)` — целые 0…10¹² (и отрицательные: «минус»).
      Правило: перед **жүз/мың** ведущее «бір» опускается; перед миллион/миллиард —
      сохраняется. 0 → «нөл».
- [ ] Десятичные (D2) и разрядные пробелы («1 000 000» → одно число).
- [ ] `%` → «пайыз»; `₸`/`тг`/`тенге` → «теңге» (базовый набор единиц).
- [ ] `expand_numbers_kk(text)` — регекс-замена «голых» чисел (без хвостов-суффиксов)
      на количественные; интеграция в `tts_service` посегментно.
- [ ] pytest: табличные кейсы (десятки/сотни/тысячи/миллионы, 0, десятичные, %,
      валюта, число в предложении, число в конце предложения).

**Definition of Done**: «Бізде 1250 кітап және 5% жеңілдік бар.» синтезируется как
«…бір мың екі жүз елу кітап және бес пайыз жеңілдік…»; «3.14» не рвёт предложение.

⛔ **Переход к Этапу 2 — после одобрения.**

## Этап 2 — Порядковые числительные + падежные окончания

**Цель**: екі→екінші и корректные окончания на числах (10-ға → онға).

**Объём**:
- [ ] `ordinal_kk(n)` — строим количественное, **последнее слово** заменяем по
      таблице порядковых основ (см. приложение; учитывает нерегулярности
      қырық→қырқыншы, жиырма→жиырмасыншы). Составные: «21-ші» → «жиырма бірінші».
- [ ] Распознавание **явных маркеров** в тексте: хвост `-ші/-шы/-ыншы/-інші/-нші/
      -ншы` после числа → порядковое (суффикс из текста снимаем).
- [ ] `attach_case(word, case)` — приклеивание падежных окончаний с сингармонизмом
      и ассимиляцией (дательный/местный/исходный/винительный/родительный/творит.).
      Хвосты вида «10-ға», «5-тен», «2015-жылы», «3-ке».
- [ ] Контекстные порядковые по ключевым словам (жыл, ғасыр, сынып, класс, бет,
      тарау, №/нөмір) — по решению D1. Список настраиваемый.
- [ ] pytest: порядковые по диапазонам, все падежи × классы окончаний, составные,
      маркеры из текста.

**Definition of Done**: «5-сынып», «2-ші орын», «10-ға дейін», «2024 жыл»
читаются корректно (бесінші сынып; екінші орын; онға дейін; …жиырма төртінші жыл).

⛔ **Переход к Этапу 3 — после одобрения.**

## Этап 3 — Даты, время, диапазоны, телефоны

**Цель**: осмысленное чтение дат/времени и спец-форматов.

**Объём**:
- [ ] Даты: `dd.mm.yyyy`, `dd.mm`, «d <месяц>» → «<день-порядк.> <месяц>
      [<год-порядк.> жыл]». Таблица месяцев (қаңтар…желтоқсан).
- [ ] Время: `HH:MM` → «<сағат> сағат <минут> минут» (варианты «жарты», «ширек» —
      опционально).
- [ ] Диапазоны: «5-10», «2020-2024» → «бестен онға дейін» / контекст.
- [ ] Телефоны/длинные коды → позначно или группами.
- [ ] pytest на каждый формат + смешанные предложения.

**Definition of Done**: «Кездесу 05.05.2024, сағат 14:30» читается как
«…бесінші мамыр екі мың жиырма төртінші жыл, сағат он төрт сағат отыз минут».

⛔ **Переход к Этапу 4 — после одобрения.**

## Приложение A. Количественные (основа)
```
единицы: 1 бір · 2 екі · 3 үш · 4 төрт · 5 бес · 6 алты · 7 жеті · 8 сегіз · 9 тоғыз
десятки: 10 он · 20 жиырма · 30 отыз · 40 қырық · 50 елу · 60 алпыс · 70 жетпіс · 80 сексен · 90 тоқсан
разряды: 100 жүз · 1 000 мың · 10⁶ миллион · 10⁹ миллиард · 10¹² триллион · 0 нөл
```

## Приложение B. Порядковые основы (последнее слово)
```
бір→бірінші екі→екінші үш→үшінші төрт→төртінші бес→бесінші
алты→алтыншы жеті→жетінші сегіз→сегізінші тоғыз→тоғызыншы
он→оныншы жиырма→жиырмасыншы отыз→отызыншы қырық→қырқыншы елу→елуінші
алпыс→алпысыншы жетпіс→жетпісінші сексен→сексенінші тоқсан→тоқсаныншы
жүз→жүзінші мың→мыңыншы миллион→миллионыншы миллиард→миллиардыншы
```
> Нерегулярности (қырық→қырқыншы с выпадением гласной, жиырма→жиырмасыншы)
> храним таблицей, а не общим правилом.

## Приложение C. Падежные окончания (сингармонизм + ассимиляция)
Задний ряд: гласные **а о ұ ы**; передний ряд: **ә ө ү е і**.

| Падеж | После гласной/звонкой | После глухой | После н/ң/м |
|---|---|---|---|
| Дательный (барыс) | -ға/-ге | -қа/-ке | -ға/-ге |
| Местный (жатыс) | -да/-де | -та/-те | -да/-де |
| Исходный (шығыс) | -дан/-ден | -тан/-тен | -нан/-нен |
| Винительный (табыс) | -ны/-ні (после гласн.), -ды/-ді (после звонк.) | -ты/-ті | -ды/-ді |
| Родительный (ілік) | -ның/-нің / -дың/-дің | -тың/-тің | -ның/-нің |
| Творительный (көмектес) | -мен | -пен | -бен |

> ⚠️ Таблица — рабочая основа; окончания и краевые случаи (леницию қ→ғ, к→г
> перед гласной) **выверить с носителем** (D4) и закрепить тестами.
